# Security Policy

## Supported surface

This repository is currently maintained as a vertical-slice game project, so security reports should focus on:

- exposed credentials or tokens
- dangerous workflow permissions in GitHub Actions
- unsafe automation scripts in `scripts/codex/`
- supply-chain issues introduced by repository tooling

## Reporting

Please do not open a public issue for a suspected security problem.

Report security concerns privately to the repository owner through GitHub Security Advisories or a direct maintainer contact on the [DigitalHerencia GitHub profile](https://github.com/DigitalHerencia).

Include:

- a concise description of the issue
- affected paths or workflow files
- reproduction steps or proof of concept
- impact assessment

## Response expectations

- Initial triage target: within 5 business days
- Remediation target: based on severity and exploitability
- Coordinated disclosure is preferred after a fix is available
